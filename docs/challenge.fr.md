# Challenge

## Titre

**Strategic Radar x+12 - Open Data pour l’analyse prospective de l’environnement et les indicateurs de résilience stratégique**

## Résumé

Le challenge vise à mettre en place une analyse prospective de l’environnement automatisée et pilotée par les données, avec un horizon d’environ 10 à 15 ans. Des sources publiques issues des statistiques, géodonnées, climat, dangers naturels, énergie, mobilité, politique, réglementation, actualités et études doivent être intégrées, structurées et visualisées afin de rendre visibles les tendances, indicateurs de résilience et weak signals.

## Cible du hackathon

Le hackathon ne doit pas produire une plateforme complète, mais un MVP ciblé avec une ligne technique claire :

- connecter des sources de données publiques pertinentes
- réunir les données dans un modèle commun
- rendre les indicateurs et les preuves traçables
- fournir une première vue exploitable pour l’analyse et la discussion

## Ce que le challenge doit résoudre

| Aspect | Attente |
| --- | --- |
| Horizon temporel | env. 10 à 15 ans |
| Base de données | uniquement des données et documents publiquement disponibles |
| Résultat | prototype fonctionnel plutôt qu’une plateforme complète |
| Focus | observation des tendances, indicateurs de résilience, weak signals |
| Transparence | références de sources claires et flux de données documentés |

## Impact attendu

Le challenge répond au décalage entre des données ouvertes dispersées, des signaux non structurés et la nécessité pratique d’observer les évolutions de manière cohérente sur des horizons plus longs. Un bon résultat doit donc fournir non seulement une visualisation, mais aussi une traçabilité fiable.

!!! tip "Idée directrice"
    Construisez une solution end-to-end petite mais robuste : quatre sources bien documentées avec une bonne structure valent mieux qu’un ensemble large mais instable sans chaîne de preuve.

## Hors périmètre

- pas de plateforme complète d’analyse stratégique
- pas d’usage de données internes, opérationnelles ou classifiées
- pas de recherche du volume maximal de données sans modèle solide
- pas de sorties de type boîte noire sans ancrage visible dans les sources
