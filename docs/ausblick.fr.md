# Perspectives après le hackathon

## Réutilisation attendue

Les résultats doivent être examinés en interne après le hackathon. L’enjeu n’est pas seulement la démo, mais aussi la valeur potentielle pour un pilote ou une suite limitée.

## Outputs pertinents

Les résultats suivants sont particulièrement utiles pour la suite interne :

- modèle de données
- documentation des sources
- logique d’intégration
- prototype de tableau de bord
- indicateurs et visualisations ayant la plus forte valeur pilote

## Étapes possibles ensuite

| Étape | Objectif |
| --- | --- |
| Expert review | évaluer l’utilité, les limites et la qualité des données |
| Atelier de priorisation | sélectionner les cas d’usage et indicateurs les plus prometteurs |
| Travaux de suivi limités | renforcer la technique, améliorer la qualité des données et affiner la logique |
| Exploration pilote | évaluer si un pilote ciblé ou une suite supplémentaire est pertinent |

## Pourquoi cette perspective compte

Une bonne contribution de hackathon ne crée pas seulement de l’attention. Elle fournit une base solide pour les décisions ultérieures. C’est pourquoi des sources claires, un modèle de données réutilisable et un chemin d’architecture traçable sont essentiels.

!!! note "Orientation pour les équipes"
    Concevez le prototype dès le départ de manière à ce que les artefacts centraux restent utiles après l’événement : modèle de données, documentation, logique d’intégration et approche de visualisation.
